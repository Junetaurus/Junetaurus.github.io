iOS 底层原理之 Objective-C 的本质

发表于2023-04-16|更新于2023-04-16
|阅读量:

Objective-C 的本质

  • 底层实现都是 C/C++ 代码

  • Objective-C -> C/C++ -> 汇编语言 -> 机器语言

  • Objective-C 的都对象、类主要是基于 C/C++ 的结构体来实现的

clang -rewrite-objc main.m -o main.cpp

xcrun -sdk iphoneos clang -arch arm64 -rewrite-objc main.m -o main.cpp

  • NSObject 的底层实现

一个 NSObject 对象占用多少内存

  • 系统分配 16 个字节，可通过 malloc_size 函数查看

  • 对象内部只使用 8 个字节，可通过 class_getInstanceSize 函数查看（64 bit）

内存对齐（iOS 16 的倍数）

实例对象 instance

  • 通过类 alloc 出来的对象
  • 在内存中存储的信息主要包括
    • isa 指针
    • 其他成员变量（具体值）

类对象 class

  • 每个类在内存中只有一个 class 对象
  • 在内存中存储的信息主要包括
    • isa 指针
    • superclass 指针
    • 类的属性信息（@property）
    • 类的对象方法信息（instance method）
    • 类的协议信息（protocol）
    • 类的成员变量信息（ivar）

元类对象 meta-class

  • 每个类在内存中只有一个 meta-class 对象
  • 在内存中存储的信息主要包括
    • isa 指针
    • superclass 指针
    • 类的类方法信息（class method）

isa superclass

  • instance 的 isa 指向 class
  • class 的 isa 指向 meta-class
  • meta-class 的 isa 指向基类的 meta-class
  • class 的 superclass 指向父类的 class
  • meta-class 的 superclass 指向父类的 meta-class
  • 基类的 meta-calss 的 superclass 指向基类的 class

class 的结构

  • 源码下载地址：https://opensource.apple.com/tarballs/objc4/
  • class、meta-class 对象的本质结构都是 struct objc_class

KVO (Key-Value Observing)

本质：

  • 利用 runtime 动态生成一个 NSKVONotifying_xx 的子类，并且让实例对象的 isa 指向这个子类
  • 当修改实例对象的属性时，会调用 Foundation 框架里面的 _NSSetxxValueAndNotify 函数
    • willChangeValueForKey
    • 父类 setter
    • didChangeValueForKey
  • 触发 observeValueForKeyPath: ofObject: change: context:

KVC (Key-Value Codeing)

使用：

Category

  • 底层结构
  • 加载处理过程
    • 通过 Runtime 加载某个类的所有 Category 的数据
    • 把所有 Category 的方法、属性、协议数据合并到一个数组中
      • 后编译的 Category 数据，会在数组的前面
    • 将合并后的数据，插入到类原来数据的前面

load initialize

load

  • load 方法会在 runtime 加载类、分类的时候调用
  • 每个类、分类的 load 方法，只会调用一次
    • load 方法是根据方法地址直接调用，并不是 objc_msgSend 函数调用
  • 调用顺序
    • 先调用类的 load 方法
      • 先编译，先调用
      • 调用子类的 load 方法前会先调用父类的 load 方法
    • 在调用分类的 load 方法
      • 先编译，先调用

initialize

  • initialize 方法会在类第一次接收到消息的时候调用
    • 先初始化父类，在初始化子类，每个类只会初始化一次
    • initialize 是通过 objc_msgSend 函数调用
      • 如果子类没有实现 initialize 方法，会调用父类的 initialize 方法
      • 如果分类实现了 initialize 方法，会覆盖类的 initialize 方法
  • 调用顺序
    • 先调用父类的 initialize 方法，在调用子类的 initialize 方法

关联对象

  • 类中的添加的属性会自动生成
    • 成员变量
    • set get 方法声明
    • set get 方法实现
  • 分类中添加的属性会自动生成
    • set get 方法声明

API

原理

  • 关联对象存储在全局的一个 AssociationsManager 中
  • 设置关联对象为 nil，就相当于移除关联对象
  • 如果被关联对象释放，关联对象自动移除

RunLopp

  • 线程和 runloop 一一对应，保存在一个全局的字典中，key = thread，value = runloop
  • 线程创建的时候是没有 runloop 的，在第一次获取的时候才会创建，主线程在 main 函数中已经获取过。
