链接器的作用
发表于2023-04-05
链接器 链接器的最主要作用就是将符号绑定到地址上。 编译器 iOS 编写的代码是先使用编译器把代码编译成机器码，然后直接在 CPU 上执行机器码的。 采用编译器生成机器码执行的好处是效率高，缺点是调试周期长。 解释器 在运行时解释执行代码，获取一段代码后就会将其翻译成目标代码，然后在执行目标代码。 解释器执行的好处是编写调试方便，缺点是执行效率低。 LLVM Apple 使用的编译器，相比 GCC，编译速度提高了 3 倍。 编译器会对每个文件进行编译，生成 Mach-O（可执行文件）。 内置链接器会将项目中的多个 Mach-O 文件合并成一个。
iOS 架构设计
发表于2023-04-05
SOLID 原则 单一功能原则：对象功能要单一，不要在一个对象里添加很多功能。 开闭原则：扩展是开放的，修改是封闭的。 里氏替换原则：子类对象是可以替代基类对象的。 接口隔离原则：接口的用途要单一，不要在一个接口上根据不同入参实现多个功能。 依赖反转原则：方法应该依赖抽象，不要依赖实例，iOS 开发就是高层业务方法依赖于协议。
Auto Layout 基本原理
发表于2023-04-04
Auto Layout 的来历 1997年，布局算法 Cassowary 问世。 2001年，Apple 将 Cassowary 算法运用到 Auto Layout 中。 Auto Layout 的生命周期 Layout Engine 是 Auto Layout 的布局引擎，用来统一管理布局的创建、更新和销毁。 视图在布局之前，Layout Engine 会将视图，约束，优先级，固定大小通过计算转换成最终的大小和位置，每当约束发生变化，Layout Engine 就会重新计算布局，获取到布局后调用 superview.setNeedLayout()，然后触发 Deffered Layout Pass，完成后进入监听约束变化的状态。 Deferred Layout Pass 主要做容错处理，比如有些视图缺失布局或者约束不确定。 Layout Engine 从上到下调用 layoutSubviews() ，通过 Cassowary 算法计算子视图的位置，然后拷贝子视图的 frame，之后就和手写布局流程一样了。
